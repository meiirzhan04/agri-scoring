# Архитектура решения — Кейс 2: Скоринговая система субсидирования АПК

**Decentrathon 5.0 | Трек: AI inDrive | Кейс 2**

---

## Проблема

Государственные субсидии АПК распределяются по принципу «кто первый подал заявку — тот и получил». Реальная эффективность хозяйства, история использования субсидий и потенциал роста не учитываются. Результат: неэффективное использование бюджета.

## Решение

AI-система merit-based скоринга, которая ранжирует заявителей по данным и формирует shortlist для комиссии. Финальное решение всегда остаётся за человеком.

## Скоринговая формула

```
Group_score(g) = mean(normalized_features в группе g) × 100

Final_score = Σ Group_score(g) × Weight(g)
            = History×0.30 + Efficiency×0.35 + Farm×0.20 + Risk×0.15

Shortlist = {фермер | Final_score ≥ 60}
```

**Гибридная схема** (при наличии обучающих данных):
```
Final_score = 0.60 × Rule_score + 0.40 × XGBoost_probability × 100
```

## Почему не «чёрный ящик»

1. **Rule-based ядро** — веса и правила понятны и обоснованы экспертами МСХ
2. **SHAP** — для каждого фермера показывает вклад каждого из 12 признаков
3. **Человек в петле** — shortlist, не автоматическое решение
4. **Аудит-трейл** — все расчёты сохраняются в JSON

---

## Ограничения системы

- Качество скоринга зависит от полноты предоставленных данных
- Proxy-метка для обучения XGBoost может не отражать реальное "качество" фермера
- Климатические данные — на уровне региона, не конкретного поля
- Модель требует переобучения при значительном изменении структуры субсидирования
